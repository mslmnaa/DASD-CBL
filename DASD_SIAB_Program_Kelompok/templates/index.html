Dua pertanyaan, dua tingkat kesulitan

Nilai tukar harian mendekati random walk. Karena itu sistem ini jujur membedakan:

  • Angka persis besok? Sulit — model hanya unggul tipis atas baseline "besok = hari ini".
  • Naik atau turun? Bisa — model benar ~{{ info.akurasi_arah }}% (jauh di atas 50%).
Maka model juara dipilih berdasarkan akurasi arah, metrik yang paling bermakna untuk keputusan valas.

Alur sistem (MLOps sederhana)

  • Data Collection → Preprocessing (koreksi outlier, fitur return)
  • Model Training (RF, XGBoost, Linear) → Evaluasi vs baseline
  • Pemilihan model otomatis → simpan .pkl
  • Deployment (Flask di cPanel) → inferensi tanpa training ulang
  • Monitoring & retraining berkala
